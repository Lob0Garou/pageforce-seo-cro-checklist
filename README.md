# PageForce SEO/CRO Checklist

Public interactive checklist for auditing landing pages before spending more on traffic.

It is a small PageForce tool for founders, marketers, and local businesses that need to review the parts of a funnel that usually leak revenue quietly: offer clarity, trust, mobile conversion, WhatsApp handoff, CRM follow-up, SEO basics, and tracking.

## Live Demo

- GitHub Pages: https://lob0garou.github.io/pageforce-seo-cro-checklist/
- PageForce: https://pageforce.com.br/

## What It Audits

- Offer clarity and above-the-fold message.
- SEO basics for the page and local intent.
- Trust, proof, and friction points.
- Mobile experience and primary CTA visibility.
- WhatsApp handoff and response context.
- CRM, lead status, and follow-up discipline.
- Source tracking, UTMs, and conversion measurement.

## Why This Exists

Most landing pages do not fail because of one dramatic mistake. They fail through small gaps that compound:

- unclear offer;
- weak proof;
- mobile CTA hidden below the fold;
- WhatsApp conversation without context;
- lead captured but never followed up;
- traffic running without source tracking.

This checklist turns those gaps into a simple audit flow that can be used before launching ads, rebuilding a landing page, or reviewing a local-business funnel.

## How It's Built

Single-file vanilla HTML/CSS/JS — no framework, no build step, no dependencies. Checkbox state drives a live score with tiered guidance, and the whole thing deploys straight from this repo via GitHub Pages.

## Positioning

This repo is intentionally public and lightweight: it shows the PageForce approach without exposing client work or internal delivery material.
